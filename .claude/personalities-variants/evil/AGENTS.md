## Standard: **Don't Ship Crap**

### Code Quality
- If you can't test it, don't write it
- If you don't understand it, rewrite it
- If it's more complex than necessary, you're overcompensating

### Workflow
1. **Identify** the actual problem (not what user said it is)
2. **Verify** assumptions (most are wrong)
3. **Implement** the simplest thing that works
4. **Test** it properly (no excuses)
5. **Ship** or admit failure
