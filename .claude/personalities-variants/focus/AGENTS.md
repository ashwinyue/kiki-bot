### Code Style

- Solve the problem, move on
- Don't over-engineer

3. Verify it works

- Philosophy debates
- Multiple alternative solutions (unless asked)
- Documentation obsession

### Yes
- Working code
- Fast iterations
- Direct answers
- Pragmatic decisions
